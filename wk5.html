Dev Blog

Heya, this week I started learning the basics of JavaScript. So far it's been fairly simple to understand as its somewhat similar to languages I have already learnt (In particular Python lol)

Im gonna write a bit so this will be significantly longer than the others lol.

There are of course differences in both languages
Objects seems to act like dictionaries in Python, they both have a key value pair function.
However the main difference is that the "Keys" in objects are reffered as properties (cuz they are "objects")
Anyways, in order to refer to objects, you can just type the object name and the property you are refering to after a dot. For example:

var objectThing = {
color : "blue"
number : 2
Value : true
}
objectThing.color
objectThing.Value = false
objectThing.string = "Words"

objectThing

So what this will print is

'blue'
false
'Words'
{ color: 'blue', number: 2, Value: false, string: 'Words'}

As you can see its very straightforward and simple compared to python where you have to type fromkeys() or whatever

There are some other features that I like though, for example:
Theres two types of comparison operators, them being == and ===
The difference between them is that === is more strict, for example:

var one = "1"
one == 1
one === 1

The 1st comparison line (==) will print out a true value
while the 2nd comparison line (===) will print out a false value
Thats because the variable one is a string value of "1"
Since === is more strict, "1" is not the same as 1
So since == is less strict, it treats "1" and 1 as the same

Some functions are also simplified, for example if you want to check the type of variable a variable is, you use typeof
Very straightforward which I like so.

There are of course some things I dislike about JavaScript.
One such is that "and" operators and "or" operators are not words but are && and || respectively
It will be pretty weird to type them out, especially || but I think I'll get used to it at some point
Well, at least if operators are still if

Another thing to note is that it seems to follow C language syntax when it comes to conditionals and functions where you need to type {} after.
Though it seems possible to type one liners in if else statements. (Usually only applicable if there is one if else statement)

There is also other methods apart from if and else, one of them are called switch case. They act as a bunch of if else statements but they require a break statement since each case does not have a {}.
Theres also something called default that really is just the last "else" in a multiple if else block.
Overall this is an interesting method for if else and it has other applications such as cascading code by omitting the break

But that's not all, there is another method for one liner if else statement which uses ? and :
If its not obvious the ? refers to "if" and : refers to "else".
They look kinda weird to read but its shorter that if else. I don't think I will be using this method often though.
